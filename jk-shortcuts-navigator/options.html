    • JK Navigator Options

    Sites

    Manage your custom sites for which you'd like to enable the JK Navigator here:

    Add site | Restore default sites

    Explanation

    selectors: css selector(s) for the links that we want to navigate.
    paginator_selector_next: css selector for the link that opens next page of results
    paginator_selector_prev: css selector for the link that opens next page of results
    allowSubdomains: whether or not subdomains should trigger the jk navigator (www is not treated as subdomain here and will always work). docs.google.com for example has its own jk functionality and should not be broken through the one on google.com
    search_selector: css selector for the search input field
    liveUpdateElement: for pages with ajax, such as Google and Facebook, that load search results dynamically, we need to know the parent liveUpdate Element so that we can load jk at the right time when results are here.
    infiniteScroll: set to True if we have a page like Facebook or Quora that can dynamically load new links into the link list.
